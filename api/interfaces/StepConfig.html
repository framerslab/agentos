Interface StepConfig

Configuration for a single workflow step node.

Exactly one of tool, gmi, human, extension, or subgraph must be provided to specify the execution strategy. All remaining fields are optional policies.

interface StepConfig {
    tool?: string;
    gmi?: {
        instructions: string;
        executionMode?: "single_turn";
        maxTokens?: number;
    };
    human?: {
        prompt: string;
    };
    extension?: {
        extensionId: string;
        method: string;
    };
    subgraph?: CompiledExecutionGraph;
    memory?: MemoryPolicy;
    discovery?: DiscoveryPolicy;
    guardrails?: GuardrailPolicy;
    requiresApproval?: boolean;
    onFailure?: "abort" | "skip" | "retry";
    retryPolicy?: {
        maxAttempts: number;
        backoff: "exponential" | "linear" | "fixed";
        backoffMs: number;
    };
    timeout?: number;
    effectClass?: "external" | "human" | "pure" | "read" | "write";
    outputAs?: string;
    voice?: VoiceNodeConfig;
}

Optional outputAs

outputAs?: string

Optional artifact key name. When set, this step's successful output is promoted into state.artifacts[outputAs] (the value returned from invoke()). When omitted, the step's output is promoted into state.artifacts[<stepId>] by default. Use this to make your final step populate a key that matches the shape of your .returns() schema — e.g. step('publish', { tool: '...', outputAs: 'publishedTo' }) when .returns(z.object({ publishedTo: z.array(z.string()) })).
